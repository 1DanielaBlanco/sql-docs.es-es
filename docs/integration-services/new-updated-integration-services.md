**Editor del Administrador de conexiones FTP**

  Utilice el cuadro de diálogo **Editor del administrador de conexiones FTP** para especificar propiedades de conexión con un servidor FTP.

> [!IMPORTANT]
>  El administrador de conexiones FTP solo admite la autenticación anónima y la autenticación básica. No es compatible con la autenticación de Windows.

 Para obtener más información sobre el Administrador de conexiones FTP, vea [FTP Connection Manager--../../ integration-services/connection-manager/ftp-connection-manager.md).

**Opciones**

 **Nombre del servidor** proporcione el nombre del servidor FTP.

 **Puerto del servidor** especificar el número de puerto en el servidor FTP que se usará para la conexión. El valor predeterminado de esta propiedad es **21**.

 **Nombre de usuario** proporcione un nombre de usuario para acceder al servidor FTP. El valor predeterminado de esta propiedad es **anonymous**.

 **Contraseña** proporcionar la contraseña para acceder al servidor FTP.

 **Tiempo de espera (en segundos)** especificar el número de segundos que transcurrirán antes de expirar. Si el valor es **0** , indica un período de tiempo infinito. El valor predeterminado de esta propiedad es **60**.

 **Usar modo pasivo** especificar si el servidor o el cliente inicia la conexión. El servidor inicia la conexión en modo activo y el cliente en modo pasivo. El valor predeterminado de esta propiedad es **active mode**.

 **Vuelve a intentar** especificar el número de veces que la tarea intenta establecer una conexión. Un valor de **0** indica que no existe límite en el número de intentos.

 **La fragmentación de tamaño (en KB)** indique un tamaño de fragmento en kilobytes para transmitir datos.

 **Probar conexión** después de configurar el Administrador de conexiones FTP, confirme que la conexión es viable haciendo clic en **Probar conexión**.

**Aspectos destacados de esta versión**


Estos son las características nuevas más importantes de Integration Services para SQL Server 2017.

-   **Escalar horizontalmente**. Distribuir la ejecución de paquetes SSIS más fácilmente en varios equipos de trabajo y administrar ejecuciones y los trabajadores de un único equipo maestro. Para obtener más información, consulte [Integration Services escala fuera--.. / integration-services/scale-out/integration-services-ssis-scale-out.md).

Además de las mejoras de conectividad en SQL Server Integration Services Feature Pack para Azure ha agregado compatibilidad para almacén de Azure Data Lake.
