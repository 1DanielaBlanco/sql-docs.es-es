# <a name="guide-to-enhancing-privacy-and-addressing-gdpr-requirements-with-the-microsoft-sql-platform"></a>Guía para la mejora de la privacidad y direccionamiento GDPR requisitos con la plataforma de Microsoft SQL


## <a name="summary"></a>Resumen
En el 25 de mayo de 2018 la legislación de privacidad europeas es debido a surten efecto que establece una nueva barra global para derechos de privacidad, la seguridad y cumplimiento de normas. La normativa de protección de datos generales o GDPR, es fundamentalmente sobre protección y habilitar los derechos de privacidad de las personas y establece los requisitos de privacidad global estricta datos cómo personales que rige administrados y protegidos, que respeta choice individuales. 

Los clientes de Microsoft SQL que están sujetos a la GDPR, si la administración basada en la nube, las bases de datos local o ambos, se debe asegurarse de que obtenidas datos en sus sistemas de bases de datos es muy apropiado administran y protegen según los principios GDPR. Esto significa que muchos clientes que se necesitará revisar o modificar su administración de base de datos y la manipulación de los procedimientos, especialmente centrarse en la seguridad de procesamiento de datos estipulados en el GDPR.

Las tecnologías basadas en SQL de Microsoft ofrecen muchas capacidades de seguridad integradas que pueden ayudar a reducir los riesgos a los datos y mejorar la protección y la capacidad de administración de datos en el nivel de base de datos y más allá. Este documento examina estas capacidades y comparte algunos de los enfoques de Microsoft con Microsoft SQL para lograr los objetivos de privacidad de datos de la GDPR.
